# OpenERV Experimental Testing Suite

This directory contains a suite of diagnostic and experimental scripts for developers. These are intended for manual execution during hardware bring-up, troubleshooting, or performance characterization.

## 1. Hardware Verification (`/hardware`)
These scripts test low-level Pico W and PCB functionality. Run them by uploading to the Pico and executing via REPL or renaming to `main.py`.

- **`i2c_scanner.py`**: Scans the I2C bus (GP18/GP19) and reports addresses of found devices (e.g., SDP810).
- **`onboard_led_blink.py`**: Simple blink test for the internal Pico W LED.
- **`rgb_led_cycle.py`**: Cycles the external RGB status LEDs through Red, Green, and Blue.
- **`button_led_toggle.py`**: Toggles the onboard LED when the physical button (GP20) is pressed.
- **`rgb_button_cycle.py`**: Cycles RGB colors each time the button (GP20) is pressed.
- **`pin_toggle_test.py`**: Rapidly toggles GP1 to check for continuity or signal integrity.

## 2. System Applications (`/apps`)
Standalone MicroPython applications for specific experimental tasks.

- **`thermistor_logger_sd.py`**: Records high-precision temperature data from thermistors to an attached SD card.
- **`thermistor_logger_console.py`**: Same as above, but outputs data to the Serial Console (for units without SD cards).
- **`anemometer_diagnostic.py`**: Implements a "hot-wire" anemometer using heated thermistors to measure airflow proxy.

## 3. Data Analysis (`/analysis`)
Python scripts intended to run on a **PC** (requires `numpy` and `matplotlib`) to process logs generated by the system apps.

- **`merge_logs.py`**: Combines multiple JSON sample files into a single `samplescombined.json`.
- **`plot_scatter.py`**: Generates a scatter plot of temperature vs. time.
- **`2d plot working from thermistor data.py`**: Line plots for specific data trends.
- **`graph thermistor vals.py`**: 3D visualization of sensor data.

## Usage Guide
1. **Choose a test**: Identify the script that matches your hardware component.
2. **Deploy**: Upload the script to the Raspberry Pi Pico W.
3. **Execute**: Run the script using your IDE (e.g., Thonny) or rename it to `main.py` for a cold boot test.
4. **Observe**: Check the serial output or physical LED/Fan response for the expected behavior.
